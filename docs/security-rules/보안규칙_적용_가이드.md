# 최종 보안 규칙 적용 가이드

## 📋 최종 보안 규칙 파일

**파일명**: `Firestore_보안규칙_최종본.txt`

이 규칙은 다음 기능을 포함합니다:
- ✅ 사용자 정보: 본인만 읽기/수정, 관리자는 모든 권한
- ✅ A/S 게시판: 공개 읽기, 인증된 사용자만 작성
- ✅ 직원 게시판: 임직원만 접근
- ✅ 활동사진첩: 인증된 사용자만 접근

---

## 🚀 적용 방법

### 1단계: Firebase Console 접속

1. 웹 브라우저에서 **https://console.firebase.google.com** 접속
2. Google 계정으로 로그인
3. 프로젝트 **"hanarooa-f227d"** 선택

---

### 2단계: Firestore Database > 규칙 탭

1. **왼쪽 사이드바에서 "Firestore Database" 클릭**
2. **상단 탭에서 "규칙" 선택**
3. 현재 규칙이 표시된 편집기 확인

---

### 3단계: 규칙 교체

1. **`Firestore_보안규칙_최종본.txt` 파일 열기**
2. **전체 내용 복사** (Ctrl+A, Ctrl+C 또는 Cmd+A, Cmd+C)
3. **Firebase Console의 규칙 편집기에서:**
   - 전체 선택 (Ctrl+A 또는 Cmd+A)
   - 삭제
   - 붙여넣기 (Ctrl+V 또는 Cmd+V)

---

### 4단계: 규칙 게시 (매우 중요!)

1. 규칙 편집기 하단의 **"게시"** 또는 **"Publish"** 버튼 클릭
2. 확인 대화상자에서 **"게시"** 클릭
3. **"규칙이 게시되었습니다"** 메시지 확인
4. 규칙 탭 상단에 **"게시됨"** 상태 확인
5. 마지막 게시 시간 확인

⚠️ **주의**: 규칙을 편집만 하고 게시하지 않으면 적용되지 않습니다!

---

### 5단계: 규칙 확인

1. 규칙 탭 상단에 **"게시됨"** 상태 확인
2. 규칙 편집기에 다음이 포함되어 있는지 확인:
   - `match /users/{userId}`
   - `allow read: if request.auth != null && (request.auth.uid == userId || isAdmin())`
   - `function isAdmin()`
   - `function isEmployee()`

## 📋 보안 규칙 설명

### 사용자 정보 (`/users/{userId}`)

**읽기:**
- 본인 정보만 읽기 가능
- 관리자는 모든 사용자 정보 읽기 가능

**생성:**
- 본인만 회원가입 시 생성 가능

**수정:**
- 본인만 수정 가능
- 관리자는 모든 사용자 정보 수정 가능

**삭제:**
- 관리자만 가능

---

### A/S 게시판 (`/asPosts/{document}`)

**읽기:**
- 모든 사용자 가능 (공개 게시판)

**생성:**
- 인증된 사용자만 가능

**수정/삭제:**
- 본인이 작성한 게시글만 가능
- 관리자는 모든 게시글 수정/삭제 가능

---

### 직원 게시판 (`/staffPosts/{board}`)

**읽기/쓰기:**
- 임직원만 가능
- 관리자도 가능

---

### 활동사진첩 (`/activityPhotos/{photoId}`)

**읽기:**
- 인증된 사용자만 가능

**생성:**
- 인증된 사용자만 가능

**수정/삭제:**
- 본인이 업로드한 사진만 가능
- 관리자는 모든 사진 수정/삭제 가능

---

## ✅ 적용 확인

### 규칙 시뮬레이터로 테스트

1. **Firebase Console > Firestore Database > 규칙 > 시뮬레이터 탭**

2. **테스트 1: 본인 정보 읽기**
   - 위치: `users/사용자UID`
   - 읽기 선택
   - 인증됨: 체크
   - 필드: `request.auth.uid`
   - 문자열: `사용자UID`
   - 실행 → **허용됨** 확인

3. **테스트 2: 다른 사용자 정보 읽기 (일반 사용자)**
   - 위치: `users/다른사용자UID`
   - 읽기 선택
   - 인증됨: 체크
   - 필드: `request.auth.uid`
   - 문자열: `일반사용자UID` (다른 UID)
   - 실행 → **거부됨** 확인 (정상)

4. **테스트 3: 다른 사용자 정보 읽기 (관리자)**
   - 위치: `users/다른사용자UID`
   - 읽기 선택
   - 인증됨: 체크
   - 필드: `request.auth.uid`
   - 문자열: `관리자UID`
   - 실행 → **허용됨** 확인 (관리자는 모든 사용자 정보 읽기 가능)

## 🔍 문제 해결

### 문제 1: 규칙 게시 후에도 오류 발생

**확인 사항:**
1. 규칙이 "게시됨" 상태인지 확인
2. 브라우저 캐시 삭제
3. 시크릿 모드에서 테스트

### 문제 2: 규칙 문법 오류

**확인 사항:**
1. 규칙 편집기 하단에 빨간색 오류 표시 확인
2. 모든 중괄호 `{}`가 닫혀있는지 확인
3. 모든 따옴표가 올바르게 닫혀있는지 확인

### 문제 3: 헬퍼 함수 오류

**확인 사항:**
1. `function isAdmin()` 정의 확인
2. `function isEmployee()` 정의 확인
3. 함수 호출 시 괄호 `()` 포함 확인
