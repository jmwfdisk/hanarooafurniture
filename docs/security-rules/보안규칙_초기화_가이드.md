# Firestore 보안 규칙 초기화 및 단계별 설정 가이드

## 🎯 목표
보안 규칙을 완전히 초기화하고, 단계별로 필요한 권한만 추가하여 문제를 해결합니다.

## 📋 단계별 설정 방법

### 1단계: 가장 기본적인 규칙으로 시작 (로그인 테스트용)

**목적**: 로그인한 사용자가 모든 문서를 읽고 쓸 수 있도록 허용하여 로그인이 작동하는지 확인

#### 적용 방법:
1. Firebase Console > Firestore Database > 규칙 탭
2. `Firestore_보안규칙_1단계_기본.txt` 파일 내용 복사
3. 규칙 편집기에 붙여넣기
4. **"게시" 버튼 클릭**
5. 웹사이트에서 로그인 테스트

#### 규칙 내용:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 로그인한 사용자는 모든 문서 읽기/쓰기 가능
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

#### 테스트:
- ✅ 로그인이 성공하는지 확인
- ✅ 콘솔에 `permission-denied` 오류가 사라지는지 확인

---

### 2단계: A/S 게시판만 공개 (일반 개선)

**목적**: A/S 게시판은 비로그인 사용자도 접근할 수 있도록 설정

#### 적용 방법:
1. 1단계에서 로그인이 작동하는 것을 확인한 후
2. Firebase Console > Firestore Database > 규칙 탭
3. `Firestore_보안규칙_2단계_개선.txt` 파일 내용 복사
4. 규칙 편집기에 붙여넣기
5. **"게시" 버튼 클릭**
6. 웹사이트에서 로그인 및 A/S 게시판 테스트

#### 규칙 내용:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // A/S 게시판 - 모든 사용자 접근 가능
    match /asPosts/{document} {
      allow read, write: if true;
    }

    // 나머지 모든 문서 - 로그인한 사용자만
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

---

### 3단계: 세부 권한 설정 (최종)

**목적**: 각 컬렉션별로 세부 권한 설정

#### 적용 방법:
1. 2단계에서 모든 기능이 작동하는 것을 확인한 후
2. Firebase Console > Firestore Database > 규칙 탭
3. `Firestore_보안규칙_3단계_최종.txt` 파일 내용 복사
4. 규칙 편집기에 붙여넣기
5. **"게시" 버튼 클릭**
6. 모든 기능 테스트

#### 규칙 내용:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // A/S 게시판 - 모든 사용자 접근 가능
    match /asPosts/{document} {
      allow read, write: if true;
    }

    // 직원 게시판
    match /staffPosts/{board} {
      allow read, write: if request.auth != null;
    }

    // 사용자 정보
    match /users/{userId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null && request.auth.uid == userId;
      allow update, delete: if request.auth != null;
    }

    // 활동사진첩
    match /activityPhotos/{photoId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow delete: if request.auth != null && 
        resource.data.uploadedBy == request.auth.uid;
    }
  }
}
```

## ⚠️ 중요 사항

### 각 단계마다 반드시 확인할 것:
1. ✅ Firebase Console에서 **"게시" 버튼 클릭**
2. ✅ **"규칙이 게시되었습니다"** 메시지 확인
3. ✅ 규칙 탭 상단에 **"게시됨"** 상태 확인
4. ✅ 브라우저 완전 새로고침 (Ctrl+Shift+R)
5. ✅ 웹사이트에서 기능 테스트

### 문제가 발생하면:
- 이전 단계로 돌아가서 다시 테스트
- 브라우저 콘솔(F12)에서 오류 메시지 확인
- Firebase Console에서 규칙 게시 상태 확인
