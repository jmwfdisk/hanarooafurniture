## 2단계: Storage 메뉴 확인

1. Firebase Console 왼쪽 사이드바에서 **"Storage"** 메뉴 클릭
   - 메뉴가 보이지 않으면 "빌드(Build)" 섹션을 펼쳐보세요

   - 이 버튼을 클릭하여 Storage를 활성화합니다

## 3단계: Storage 활성화 (처음 사용하는 경우)

### 3-1. Storage 시작하기 클릭

1. "Storage 시작하기" 버튼 클릭
2. 다음 화면이 나타납니다

### 3-2. 보안 규칙 선택

**옵션 1: 테스트 모드 (무료 사용 권장)**
- ✅ "테스트 모드에서 시작" 선택
- 💰 무료 티어로 사용 가능
- ⚠️ 주의: 모든 사용자가 읽기/쓰기 가능 (A/S 게시판 용도로 적합)
- "다음" 클릭

**옵션 2: 프로덕션 모드**
- ✅ "프로덕션 모드에서 시작" 선택
- 🔒 인증된 사용자만 접근 가능
- 💰 무료 티어로 사용 가능
- "다음" 클릭

**💡 추천: 테스트 모드 선택**
- A/S 게시판은 모든 사용자가 파일을 업로드해야 하므로 테스트 모드가 적합
- 무료 티어로 충분히 사용 가능

### 3-3. 위치 선택

1. **Cloud Storage 위치 선택**
   - 권장: `asia-northeast3` (서울) - 한국 사용자에게 가장 빠름
   - 또는 `us-central` (미국 중부) - 기본값
2. "완료" 또는 "Done" 클릭
3. Storage 활성화 완료! (몇 초 소요)

## 4단계: Storage 활성화 확인

Storage가 활성화되면 다음 화면이 표시됩니다:

### ✅ 활성화된 경우:
- **"파일" 탭**: 업로드된 파일 목록 표시 (현재는 비어있음)
- **"규칙" 탭**: 보안 규칙 설정 가능
- **"사용량" 탭**: 저장 공간 및 다운로드 사용량 확인

### ❌ 활성화되지 않은 경우:
- "Storage 시작하기" 버튼이 계속 표시됨
- 위의 3단계를 다시 진행하세요

---

## 5단계: 보안 규칙 설정 (중요!)

### 5-1. 규칙 탭으로 이동

1. Storage 페이지에서 **"규칙"** 탭 클릭
2. 현재 규칙이 표시됩니다

### 5-2. 규칙 수정

**테스트 모드인 경우:**
```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read, write: if request.time < timestamp.date(2025, 12, 31);
    }
  }
}
```

**프로덕션 모드로 변경 (권장):**
```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    // A/S 파일 폴더
    match /as-files/{fileName} {
      // 모든 사용자가 읽기 가능
      allow read: if true;
      // 모든 사용자가 쓰기 가능 (필요시 인증 추가 가능)
      allow write: if true;
    }
    
    // 다른 경로는 접근 불가
    match /{allPaths=**} {
      allow read, write: if false;
    }
  }
}
```

### 5-3. 규칙 게시

1. 규칙 코드를 입력한 후
2. **"게시"** 또는 "Publish" 버튼 클릭
3. 확인 메시지 표시

---

## 6단계: 테스트 업로드 (선택사항)

### 6-1. 수동 테스트

1. Storage 페이지의 **"파일"** 탭으로 이동
2. **"파일 업로드"** 버튼 클릭
3. 작은 테스트 파일 선택 (예: 이미지 파일)
4. 업로드 완료 확인

### 6-2. 웹사이트에서 테스트

1. A/S 신청 페이지로 이동
2. A/S 신청 폼 작성
3. 파일 첨부
4. 등록 버튼 클릭
5. Firebase Console의 Storage > 파일 탭에서 업로드된 파일 확인

---

## 문제 해결

### ❌ "Storage 시작하기" 버튼이 계속 나타나는 경우

**원인:**
- Storage가 아직 활성화되지 않음
- Firebase 프로젝트에 Storage 서비스가 추가되지 않음

**해결:**
1. 3단계를 다시 진행
2. 브라우저 새로고침
3. 다른 브라우저에서 시도

### ❌ 파일 업로드가 실패하는 경우

**확인 사항:**
1. Storage 보안 규칙 확인
   - `as-files/{fileName}` 경로에 대한 읽기/쓰기 권한 확인
2. 브라우저 콘솔 확인
   - F12 키를 눌러 개발자 도구 열기
   - Console 탭에서 오류 메시지 확인
3. Firebase 설정 확인
   - `AS.html` 파일의 `firebaseConfig` 값이 올바른지 확인

### ❌ "Permission denied" 오류

**원인:**
- 보안 규칙이 너무 엄격함

**해결:**
- Storage > 규칙 탭에서 위의 5-2 단계 규칙 적용

## Firebase 무료 티어 (Spark Plan) 정보

### ✅ 무료로 사용 가능한 기능

**Firebase Storage 무료 한도:**
- **저장 용량**: 5GB (무료)
- **다운로드**: 1GB/일 (무료)
- **업로드**: 1GB/일 (무료)
- **작업 수**: 20,000회/일 (무료)

**기타 무료 서비스:**
- Firebase Authentication: 무제한
- Firebase Realtime Database: 1GB 저장, 10GB/월 전송
- Firebase Hosting: 10GB 저장, 360MB/일 전송

### 💰 유료 플랜 (Blaze Plan) 전환 시

**무료 티어 초과 시:**
- 초과분에 대해서만 과금
- 예: 6GB 저장 시 → 1GB만 과금 (약 $0.026/GB/월)
- 2GB 다운로드 시 → 1GB만 과금 (약 $0.12/GB)

**Blaze Plan 장점:**
- 무료 티어 한도 포함
- 사용한 만큼만 과금 (Pay as you go)
- 더 많은 기능 사용 가능

### 📊 사용량 모니터링

1. Firebase Console > Storage > 사용량 탭
2. 실시간 사용량 확인 가능
3. 무료 한도 초과 시 알림 설정 가능:
   - 프로젝트 설정 > 사용량 및 청구
   - 예산 알림 설정

### ⚠️ 무료 티어 사용 시 주의사항

1. **파일 크기 제한**
   - 현재 코드: 10MB 제한 (무료 티어에 적합)
   - 더 큰 파일이 필요하면 유료 플랜 고려

2. **사용량 관리**
   - 정기적으로 사용량 확인
   - 불필요한 파일 삭제
   - 오래된 파일 정리

3. **비용 예방**
   - 무료 한도 내에서 사용
   - 예산 알림 설정
   - 사용량 대시보드 모니터링

### 🎯 무료 티어로 충분한 경우

**A/S 신청 게시판 용도로는 무료 티어로 충분합니다:**
- 평균 파일 크기: 1-2MB
- 일일 신청 건수: 10-20건 가정
- 예상 사용량:
  - 저장: 약 100-200MB/월 (5GB 한도 내)
  - 다운로드: 약 50-100MB/일 (1GB 한도 내)
  - ✅ 무료 티어로 충분!

### 💡 비용 절감 팁

1. **이미지 최적화**
   - 업로드 전 이미지 압축
   - 불필요한 메타데이터 제거

2. **파일 자동 삭제**
   - 오래된 파일 정리 (예: 1년 이상)
   - Cloud Functions로 자동화 가능

3. **캐싱 활용**
   - 자주 조회되는 파일은 CDN 캐싱
