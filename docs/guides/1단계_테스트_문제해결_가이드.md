# 1단계 테스트 문제 해결 가이드

## 🚨 현재 문제
- Firebase Auth 로그인은 성공 ✅
- Firestore에서 사용자 정보 조회 실패 ❌
- 오류: `permission-denied`

---

## ✅ 해결 방법 (단계별)

### 방법 1: 테스트용 규칙 적용 (가장 확실한 방법)

**파일**: `Firestore_보안규칙_1단계_테스트용.txt`

이 규칙은 **모든 로그인 사용자가 모든 사용자 정보를 읽을 수 있도록** 허용합니다.
이렇게 하면 로그인이 작동하는지 확실하게 확인할 수 있습니다.

#### 적용 단계:

1. **Firebase Console 접속**
   - https://console.firebase.google.com
   - 프로젝트 "hanarooa-f227d" 선택

2. **Firestore Database > 규칙 탭**
   - 왼쪽 메뉴: "Firestore Database"
   - 상단 탭: "규칙"

3. **규칙 교체**
   - `Firestore_보안규칙_1단계_테스트용.txt` 파일 열기
   - 전체 내용 복사 (Ctrl+A, Ctrl+C)
   - Firebase Console 규칙 편집기에서:
     - 전체 선택 (Ctrl+A)
     - 삭제
     - 붙여넣기 (Ctrl+V)

4. **게시 버튼 클릭** ⚠️ **중요!**
   - 하단의 **"게시"** 또는 **"Publish"** 버튼 클릭
   - 확인 대화상자에서 **"게시"** 클릭
   - **"규칙이 게시되었습니다"** 메시지 확인

5. **브라우저 완전 새로고침**
   - Windows: `Ctrl + Shift + R`
   - Mac: `Cmd + Shift + R`
   - 또는 시크릿 모드에서 테스트

6. **로그인 재시도**
   - 이메일과 비밀번호 입력
   - 로그인 버튼 클릭

7. **결과 확인**
   - ✅ 성공: "Firestore 조회 성공" 메시지 표시
   - ❌ 실패: 여전히 `permission-denied` 오류

---

### 방법 2: 사용자 문서 존재 확인

테스트용 규칙을 적용해도 여전히 오류가 발생하면, **사용자 문서가 Firestore에 없을 수 있습니다**.

#### 확인 단계:

1. **Firebase Console > Firestore Database > 데이터 탭**
   - 왼쪽 메뉴: "Firestore Database"
   - 상단 탭: "데이터"

2. **users 컬렉션 확인**
   - `users` 컬렉션이 있는지 확인
   - UID `jrSeEHFpXiPJKymdRuYDLhhjHxk1`에 해당하는 문서가 있는지 확인

3. **문서가 없는 경우**

   **옵션 A: 회원가입 다시 진행**
   - 회원가입 폼 작성
   - 회원가입 완료 후 Firestore에 문서가 생성되는지 확인

   **옵션 B: 관리자가 수동으로 생성**
   - Firebase Console > Firestore Database > 데이터
   - `users` 컬렉션 선택
   - "문서 추가" 클릭
   - 문서 ID: `jrSeEHFpXiPJKymdRuYDLhhjHxk1` (로그인한 사용자의 UID)
   - 필드 추가:
     ```
     uid: "jrSeEHFpXiPJKymdRuYDLhhjHxk1"
     email: "사용자이메일@example.com"
     username: "사용자명"
     name: "이름"
     userType: "general"
     status: "approved"
     isAdmin: false
     createdAt: "2025-01-XX..."
     ```

### 방법 3: 규칙 시뮬레이터로 테스트

Firebase Console의 규칙 시뮬레이터를 사용하여 규칙이 정상 작동하는지 확인할 수 있습니다.

1. **Firebase Console > Firestore Database > 규칙 > 시뮬레이터 탭**

2. **테스트 설정 입력**
   - **위치**: `users/jrSeEHFpXiPJKymdRuYDLhhjHxk1`
   - **읽기** 선택
   - **인증된 사용자**: UID에 `jrSeEHFpXiPJKymdRuYDLhhjHxk1` 입력

3. **실행 버튼 클릭**

4. **결과 확인**
   - ✅ **허용됨**: 규칙이 정상 작동
   - ❌ **거부됨**: 규칙에 문제가 있음

## 📋 체크리스트

다음 항목을 순서대로 확인하세요:

### 1단계: 규칙 확인
- [ ] Firebase Console에서 규칙이 "게시됨" 상태
- [ ] 규칙에 `users/{userId}` 경로가 포함되어 있음
- [ ] 규칙에 `allow read: if request.auth != null;` 포함 (테스트용)
- [ ] 마지막 게시 시간이 방금 전인지 확인

### 2단계: 사용자 문서 확인
- [ ] Firestore에 `users` 컬렉션이 존재함
- [ ] UID `jrSeEHFpXiPJKymdRuYDLhhjHxk1`에 해당하는 문서가 있음
- [ ] 문서에 필수 필드가 모두 있음 (uid, email, username 등)

### 3단계: 브라우저 확인
- [ ] 브라우저 캐시를 완전히 삭제했음
- [ ] 시크릿 모드에서 테스트했음
- [ ] 다른 브라우저에서 테스트했음

### 4단계: 로그인 테스트
- [ ] Firebase Auth 로그인 성공 확인
- [ ] 콘솔에 "Firestore 사용자 정보 조회 중..." 메시지 확인
- [ ] 콘솔에 "Firestore 조회 성공" 메시지 확인
- [ ] `permission-denied` 오류가 사라졌는지 확인

## 🔍 문제별 해결 방법

### 문제 1: 규칙 게시 후에도 오류 발생

**원인:**
- 브라우저 캐시
- 규칙이 제대로 게시되지 않음

**해결:**
1. 시크릿 모드에서 테스트
2. 규칙 탭에서 "게시됨" 상태 재확인
3. 규칙 문법 오류 확인 (편집기 하단에 빨간색 오류 표시)

### 문제 2: 사용자 문서가 없음

**원인:**
- 회원가입 시 문서가 생성되지 않음
- 문서가 삭제됨

**해결:**
1. 회원가입 다시 진행
2. 또는 관리자가 수동으로 문서 생성

### 문제 3: UID가 일치하지 않음

**원인:**
- Firebase Auth UID와 Firestore 문서 ID가 다름

**해결:**
1. 콘솔에서 로그인한 사용자의 UID 확인
2. Firestore에서 해당 UID의 문서 확인
3. 문서 ID가 UID와 정확히 일치하는지 확인

## ✅ 성공 확인

다음이 모두 확인되면 성공입니다:

- ✅ 콘솔에 `permission-denied` 오류 없음
- ✅ "Firestore 조회 성공" 메시지 표시
- ✅ 로그인 성공 알림 표시
- ✅ 사용자 정보가 정상적으로 로드됨

## 🆘 여전히 문제가 해결되지 않는 경우

1. **Firebase Console > Firestore Database > 규칙 > 시뮬레이터** 사용
   - 정확한 오류 원인 파악

2. **콘솔 오류 메시지 전체 복사**
   - F12 > Console 탭
   - 오류 메시지 전체 복사하여 확인

3. **Firebase 지원팀 문의**
   - 문제가 계속되면 Firebase 지원팀에 문의
