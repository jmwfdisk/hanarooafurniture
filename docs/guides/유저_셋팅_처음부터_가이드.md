## 1. Firebase Console에서 사용자 문서 생성

### 1단계: Firebase Console 접속

1. 웹 브라우저에서 **https://console.firebase.google.com** 접속
2. Google 계정으로 로그인
3. 프로젝트 **"hanarooa-f227d"** 선택

---

### 2단계: Authentication에서 사용자 생성

1. **왼쪽 사이드바에서 "Authentication" 클릭**
2. **상단 탭에서 "Users" 선택**
3. **"+ 사용자 추가" 또는 "Add user" 버튼 클릭**

#### 사용자 추가 방법 A: 이메일/비밀번호로 생성

1. **이메일**: 사용자 이메일 주소 입력 (예: `admin@example.com`)
2. **비밀번호**: 비밀번호 입력 (최소 6자)
3. **"사용자 추가" 또는 "Add user" 버튼 클릭**
4. **생성된 사용자의 UID 복사** (중요!)

**UID 확인 방법:**
- 사용자 목록에서 생성된 사용자 클릭
- "User UID" 필드에서 UID 복사
- 예: `jrSeEHFpXiPJKymdRuYDLhhjHxk1`

---

### 3단계: Firestore에 사용자 문서 생성

1. **왼쪽 사이드바에서 "Firestore Database" 클릭**
2. **상단 탭에서 "데이터" 선택**
3. **"+ 컬렉션 시작" 또는 "Start collection" 클릭**

#### 컬렉션 생성 (처음인 경우)

1. **컬렉션 ID**: `users` 입력
2. **"다음" 또는 "Next" 클릭**

#### 문서 생성

1. **문서 ID**: **Authentication에서 복사한 UID 입력** (중요!)
   - 예: `jrSeEHFpXiPJKymdRuYDLhhjHxk1`
   - **자동 생성하지 말고 UID를 직접 입력하세요!**

2. **필드 추가**: 아래 필드들을 하나씩 추가

---

## 2. 관리자 계정 설정

### 관리자 계정 필드 설정

**문서 ID**: Authentication에서 생성한 관리자 UID

**필수 필드:**

1. **uid** (문자열)
   - 값: 문서 ID와 동일 (예: `jrSeEHFpXiPJKymdRuYDLhhjHxk1`)

2. **email** (문자열)
   - 값: 관리자 이메일 주소 (예: `admin@example.com`)

3. **username** (문자열)
   - 값: 관리자 사용자명 (예: `admin`)

4. **name** (문자열)
   - 값: 관리자 이름 (예: `관리자`)

5. **userType** (문자열)
   - 값: `employee` (임직원으로 설정)

6. **status** (문자열)
   - 값: `approved` (승인됨)

7. **isAdmin** (불리언)
   - 값: `true` (관리자)

8. **createdAt** (문자열)
   - 값: 현재 날짜/시간 (예: `2025-01-15T12:00:00.000Z`)
   - 형식: `YYYY-MM-DDTHH:mm:ss.sssZ`

9. **phone** (문자열, 선택사항)
   - 값: 전화번호 (예: `010-1234-5678`)

---

### 관리자 계정 생성 예시

**문서 ID**: `jrSeEHFpXiPJKymdRuYDLhhjHxk1`

**필드 목록:**
```
uid: "jrSeEHFpXiPJKymdRuYDLhhjHxk1" (string)
email: "admin@example.com" (string)
username: "admin" (string)
name: "관리자" (string)
userType: "employee" (string)
status: "approved" (string)
isAdmin: true (boolean)
createdAt: "2025-01-15T12:00:00.000Z" (string)
phone: "010-1234-5678" (string) - 선택사항
```

## ⚠️ 중요 사항

### 1. 문서 ID는 반드시 UID와 일치해야 함

**올바른 예:**
- Authentication UID: `jrSeEHFpXiPJKymdRuYDLhhjHxk1`
- Firestore 문서 ID: `jrSeEHFpXiPJKymdRuYDLhhjHxk1` ✅

**잘못된 예:**
- Authentication UID: `jrSeEHFpXiPJKymdRuYDLhhjHxk1`
- Firestore 문서 ID: `user1` ❌

### 2. createdAt 형식

**올바른 형식:**
- `2025-01-15T12:00:00.000Z`
- ISO 8601 형식 사용

**생성 방법:**
- JavaScript: `new Date().toISOString()`
- 또는 현재 날짜/시간을 수동으로 입력

### 3. status 값

- `pending`: 승인 대기 (로그인 불가)
- `approved`: 승인됨 (로그인 가능)
- `rejected`: 거절됨 (로그인 불가)

**주의:** `isAdmin: true`인 경우 `status`가 `pending`이어도 로그인 가능합니다.

## 🚀 빠른 시작 가이드

### 1단계: 관리자 계정 생성 (5분)

1. Firebase Console > Authentication > Users
2. "+ 사용자 추가" 클릭
3. 이메일/비밀번호 입력
4. UID 복사

7. 필수 필드 추가 (위의 관리자 계정 예시 참고)

1. Firebase Console > Firestore Database > 규칙
2. `Firestore_보안규칙_최종본.txt` 파일 내용 복사
3. 규칙 편집기에 붙여넣기
4. "게시" 버튼 클릭

### 3단계: 테스트 (1분)

1. 시크릿 모드에서 웹사이트 접속
2. 관리자 계정으로 로그인 시도
3. 로그인 성공 확인

## 🆘 문제 해결

### 문제 1: 문서 ID를 잘못 입력함

**해결:**
- 문서 삭제 후 다시 생성
- 또는 문서 ID 변경 (Firestore는 문서 ID 변경 불가, 삭제 후 재생성 필요)

### 문제 2: 필드 타입이 잘못됨

**해결:**
- 필드 삭제 후 올바른 타입으로 다시 추가
- 예: `isAdmin`은 불리언(boolean)이어야 함

### 문제 3: 로그인이 안 됨

**확인 사항:**
- 문서 ID가 UID와 일치하는지 확인
- `status: "approved"` 또는 `isAdmin: true`인지 확인
- 보안 규칙이 게시되었는지 확인
