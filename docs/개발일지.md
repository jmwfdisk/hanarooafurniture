## 2026-06-19

### 보안 (XSS 점검 및 수정)
- **회원관리 표** (`hanaro/staff/staff.html`): 소속·이름·이메일 등 사용자 입력을 escape 처리.
  - 일반회원이 가입 시 소속/이름에 악성 스크립트를 넣어 **관리자가 회원관리를 열 때 관리자 권한 컨텍스트에서 실행**되던 권한상승형 XSS 차단.
- **게시판 제목·작성자·날짜**: A/S 게시판(`hanaro/AS/AS.html`)·직원게시판 목록/상세 모두 escape.
- **A/S 첨부파일명**: 텍스트는 HTML escape, `onclick` 속성은 JS 문자열 escape로 인젝션 차단.
- **활동사진첩 제목 수정 권한 정합성**: Firestore 규칙(본인 사진/관리자만)과 일치하도록 UI 제한. 남의 사진은 더블클릭 편집 비활성화.
- 참고(정상): Firebase API 키는 설계상 공개 키로 Firestore 규칙으로 보호됨. 보안 규칙은 catch-all deny 등 전반적으로 견고.

### 레이아웃
- **상단 navbar 반응형**: 모든 페이지(index/product/company/Gallery/AS/school)의 navbar 브레이크포인트를 1024px → **1200px**로 상향. 1025~1199px 중간 폭에서 "임직원" 버튼이 잘리던 문제 해결(햄버거 메뉴로 전환).
- **모바일 히어로**(index): `.main2-content`에 `box-sizing:border-box`, 타이틀에 `clamp()` 폰트 + 줄바꿈 허용 적용.
- **임직원 메뉴 탭**: 글자 줄바꿈 방지(`white-space:nowrap`) + 가로 스크롤로 한 줄 유지.

### 활동사진첩 (staff.html)
- 그리드 5열 고정 + 빈 칸을 점선 플레이스홀더("이미지가 없습니다")로 채움(최소 5×2).
- 사진 제목을 하단 알약형 배너로 표시. 그리드에서 더블클릭 인라인 수정(본인 사진/관리자만).
- **업로드 시 공통 제목 1개 입력** → 같은 묶음이 같은 제목 그룹으로 묶임.
- **확대 뷰어를 제목 그룹 기반으로 변경**: 같은 제목 사진만 이전/다음으로 넘김 + 위치 카운터(예: 2/3).
- **모바일 스와이프**(좌→다음, 우→이전) 지원.
- 뷰어 하단 정보(날짜·업로더·삭제) 글자 줄바꿈 방지(항목 단위 wrap).
